Planta Milenio provides 12 autocomplete endpoints used by its frontend forms to deliver real-time field suggestions as the user types. All 12 endpoints are `GET`-only, accept a single `q` query parameter containing the search string, and return a JSON array of matching results. Results are filtered using case-insensitive matching against the relevant database columns. All endpoints require an active Django session and return at most 10 results per request.

## Shared Query Parameter

All autocomplete endpoints accept the following query parameter:

<ParamField query="q" type="string" required>
  The search term. Results are filtered using case-insensitive `ILIKE` / `LIKE`
  matching against the relevant column(s). Pass an empty string to retrieve the
  default result set (behaviour varies by endpoint — see individual endpoint
  notes below).
</ParamField>

## Transport and Logistics Endpoints

These six endpoints query the external **Ceres Romana** database and are used to populate fields on the raw-material intake and dispatch forms.

### GET /autocomplete/empresa/

Searches the `empresa` table by `Empresa_Rif` or `Empresa_Nombre`. Both fields are matched independently, and any record where either field matches is returned.

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/empresa/?q=agro'
```

```json
[
  {"id": 3, "rif": "J-30000000-0", "nombre": "Agrotrack C.A."}
]
```

<ResponseField name="id" type="integer">
  Primary key of the empresa record.
</ResponseField>
<ResponseField name="rif" type="string">
  Tax identification number (RIF) of the company, e.g. `J-12345678-9`.
</ResponseField>
<ResponseField name="nombre" type="string">
  Registered company name.
</ResponseField>

### GET /autocomplete/chuto/

Searches the `Vehiculo` table by `Vehiculo_placa` (tractor-unit licence plate).

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/chuto/?q=ABC'
```

```json
[
  {"id": 1, "placa": "ABC-123", "nombre": "Camión Ford"}
]
```

<ResponseField name="id" type="integer">
  Primary key of the Vehiculo record.
</ResponseField>
<ResponseField name="placa" type="string">
  Licence plate of the tractor unit (chuto).
</ResponseField>
<ResponseField name="nombre" type="string">
  Vehicle description or model name.
</ResponseField>

### GET /autocomplete/tanque/

Searches the `Vehiculo_Remolque` table by `Vehiculo_Remolque_Placa` (tank trailer licence plate).

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/tanque/?q=XYZ'
```

```json
[
  {"placa": "XYZ-789", "nombre": "Tanque 1", "descripcion": ""}
]
```

<ResponseField name="placa" type="string">
  Licence plate of the tank trailer (tanque / remolque).
</ResponseField>
<ResponseField name="nombre" type="string">
  Short identifier or name for the trailer.
</ResponseField>
<ResponseField name="descripcion" type="string">
  Optional extended description; may be an empty string.
</ResponseField>

### GET /autocomplete/destino/

Searches the `Destino` table by `Destino_Nombre`. If `q` is empty, **all** destinations are returned rather than an empty array, making it suitable for populating an initial dropdown on page load.

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/destino/?q=norte'
```

```json
[
  {"id": 1, "nombre": "Planta Norte", "descripcion": ""}
]
```

<ResponseField name="id" type="integer">
  Primary key of the Destino record.
</ResponseField>
<ResponseField name="nombre" type="string">
  Destination name.
</ResponseField>
<ResponseField name="descripcion" type="string">
  Optional description; may be an empty string.
</ResponseField>

### GET /autocomplete/producto/

Searches the `PRODUCTO` table by `Producto_Codigo` or `Producto_Nombre`. Returns the top 10 matching products.

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/producto/?q=maiz'
```

```json
[
  {"id": 1, "codigo": "PROD-001", "nombre": "Maíz Blanco"}
]
```

<ResponseField name="id" type="integer">
  Primary key of the PRODUCTO record.
</ResponseField>
<ResponseField name="codigo" type="string">
  Internal product code, e.g. `PROD-001`.
</ResponseField>
<ResponseField name="nombre" type="string">
  Full product name.
</ResponseField>

## Visitor and Personnel Endpoints

These six endpoints query the local SQLite database via the `AccesoPersona` model and are used to populate suggestion fields on the visitor-access control form. Each endpoint targets a single field on the model, returning a plain JSON array of matching string values (not objects).

All six endpoints return at most 10 results.

### GET /autocomplete/persona/nombre/

Searches `AccesoPersona.nombre` for matching first names.

```bash
curl -b cookies.txt 'http://localhost:8000/autocomplete/persona/nombre/?q=jua'
```

```json
["Juan", "Juana"]
```

## Full curl Example

The following sequence shows a complete authenticated autocomplete flow — logging in, then querying two different endpoints:

```bash
# 1. Load the login page to receive the csrftoken cookie
curl -c cookies.txt http://localhost:8000/login/

# 2. Authenticate (replace <token> with the value from the csrftoken cookie)
curl -c cookies.txt -b cookies.txt -X POST http://localhost:8000/login/ \
  -d 'nombre=operador1&password=mypassword&csrfmiddlewaretoken=<token>'

# 3. Query a transport/logistics autocomplete endpoint
curl -b cookies.txt 'http://localhost:8000/autocomplete/empresa/?q=agro'
# Response:
# [{"id": 3, "rif": "J-30000000-0", "nombre": "Agrotrack C.A."}]

# 4. Query a visitor/personnel autocomplete endpoint
curl -b cookies.txt 'http://localhost:8000/autocomplete/persona/nombre/?q=jua'
# Response:
# ["Juan", "Juana"]
```
