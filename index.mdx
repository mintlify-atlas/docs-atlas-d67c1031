Planta Milenio is a full-stack Django 5 web application built to manage day-to-day operations at an industrial plant. It connects a local SQLite database with two external SQL Server instances (Profit ERP and Ceres Romana) to give operators a unified control panel for tracking purchase orders, vehicle movements, visitor access, and audit trails — all from a browser.

## How it works

<Steps>
  <Step title="Install and configure">
    Clone the repository, create a virtual environment, install dependencies from `requirements.txt`, and point the `DATABASES` setting at your SQL Server instances.
  </Step>
  <Step title="Create your first admin user">
    Run `CreateUser.py` from the project root to bootstrap the initial credentials — no Django admin panel required.
  </Step>
  <Step title="Log in and assign permissions">
    Sign in at `/login/`, navigate to **Usuarios**, and grant each operator access only to the modules they need (`permiso_control`, `permiso_reportes`, etc.).
  </Step>
  <Step title="Start tracking operations">
    Use **Control de Entradas** to register purchase orders, **Control de Personas** to log visitor access, and **Auditoría** to review transport records — then download PDF reports from the **Reportes** view.
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={3}>
  <Card title="Raw Material Intake" icon="truck-ramp-box">
    Validate purchase orders against the Profit ERP and register vehicle entries in the local history log.
  </Card>
  <Card title="Personnel Access" icon="id-badge">
    Log visitor entry and exit with ID, company, host, and visit duration — approve or deny in one click.
  </Card>
  <Card title="Transport Audit" icon="magnifying-glass">
    Browse and edit `orden_profit_transporte` records from Ceres Romana, with inline search and pagination.
  </Card>
  <Card title="PDF Reports" icon="file-pdf">
    Generate dated PDF exports for both material intake and personnel access using WeasyPrint.
  </Card>
  <Card title="Role-Based Access" icon="shield-halved">
    Fine-grained boolean permissions per module plus a read-only `solo_consulta` mode for auditors.
  </Card>
  <Card title="Autocomplete APIs" icon="magnifying-glass-plus">
    AJAX-driven autocomplete endpoints for companies, drivers, vehicles, products, and visitors.
  </Card>
</CardGroup>
